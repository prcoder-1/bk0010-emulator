# Джойстик БК-0010 (геймпад)

Игровой джойстик БК подключён к **параллельному порту `0177714`** (тот же порт —
принтер/covox). Активный уровень **высокий**: нажато = бит установлен, покой = 0.
Значение порта 16-битное (часть игр читают его словом; раскладка SWCorp держит
ВЛЕВО/ВВЕРХ в старшем байте). Ядро: `Board::setJoystick(uint16_t)` → возвращается
при чтении `0177714` (`ioRead`/`peekReg`).

## Ввод — без SDL, нативно через Linux

Qt6 своего модуля геймпада не имеет (`QtGamepad` был только в Qt5). SDL решили не
тянуть. `src/ui/Gamepad.{h,cpp}` читает геймпад напрямую через ядерный
joystick-интерфейс Linux — `/dev/input/js0..js3`, `linux/joystick.h` — **без внешних
зависимостей**. Неблокирующее чтение событий осей/кнопок, горячее подключение, имя
через `JSIOCGNAME`. На не-Linux — заглушка (poll() → 0). `MainWindow` опрашивает
геймпад раз в кадр. Направления берутся с крестовины **и** левого стика (оси 0/1,
хат 6/7, мёртвая зона 12000), лицевые кнопки — «огонь».

## Раскладки (у БК их исторически несколько несовместимых)

Выбор — меню **Эмуляция → Джойстик (геймпад)**; enum `Gamepad::Standard`.
Биты порта `0177714` (октально):

| Раскладка | ВВЕРХ | ВНИЗ | ВЛЕВО | ВПРАВО | кнопки «огонь» |
|-----------|-------|------|-------|--------|----------------|
| **«Стандарт»** (Джойвокс/gid, по умолч.) | `001` | `004` | `010` | `002` | 040 / 0200 / 0100 / 0400; кнопка5 = 020 |
| «Break House» (BKBTL) | `0200` | `040` | `020` | `0100` | кнопки1-4 = 001/002/004/010 |
| «SWCorp» (BKBTL) | `02000` | `040` | `01000` | `020` | кнопки1-4 = 001/002/004/010 |
| **«Клад-2»** (реверс игры) | `004` | `010` | `002` | `001` | стрельба влево/вправо = 020 / 040 |

«Стандарт» — документированная распайка (`docs/joystick` в референсе, распайка
«Джойвокс» + реализация эмулятора **gid** Manwe): ВВЕРХ=001, ВПРАВО=002, ВНИЗ=004,
ВЛЕВО=010. Не все игры ей следуют — потому раскладка выбирается.

## Как определить раскладку конкретной игры

Игры часто читают порт словом и разбирают биты; код бывает обфусцирован
(самомодификация, `CCC`-заполнители, SP как счётчик — как в Клад-2), поэтому
надёжнее **эмпирика по памяти**:

1. Найти чтение `0177714` в образе: искать слово `177714` (=`0xFFCC`, LE `CC FF`) —
   но много ложных срабатываний (это же значение = PC-смещения `JMP`/`JSR`/immediate).
   Настоящее чтение — `MOV @#177714, Rn` (013703 177714) и т.п.
2. Написать харнесс на `bkcore`: загрузить игру, ботом (Enter/цифра) войти в игру,
   затем `board.setJoystick(bit)` на N кадров и мерить, куда двигается объект —
   **по координате игрока в памяти** (напр. Клад-2 держит её в `14422`/`14426`):
   малое ±Δ = гориз., большое = верт.; знак = направление. Заблокированные
   направления (стена) не двигают — их проверяют, переставив игрока (вправо→вверх→вниз).

Пример (Клад-2): управление ←→↑↓ + стрельба РУС/ЛАТ = ровно 6 бит (0-5). Замер:
бит0→ВПРАВО(+2), бит1→ВЛЕВО(−2), бит2→ВВЕРХ(−384), бит3→ВНИЗ(подтверждено связкой).

## Тестовый стенд

К этой машине физически подключён геймпад (**RetroFlag/iBuffalo SNES**,
`/dev/input/js0`, world-readable). Для автопроверки маппинга без физических нажатий —
виртуальный контроллер SDL (`SDL_JoystickAttachVirtual`) или синтетические
`js_event`. Соответствие «направление ↔ бит» проверяется юнит-подобными харнессами.

Референсы раскладок: `emulators/БК-0010-01/bkbtl/emulator/Joystick.h` (Break House /
SWCorp), `emulators/БК-0010-01/docs/joystick/` (распайка «Джойвокс», реализация gid).
