# TODO: идеи фич отладчика и профайлера

Копилка возможностей для отладчика/профайлера. **Сделано:** символы в
дизассемблере и подсветка изменившихся регистров (см. ниже). Остальное — на
будущее; отсортировано по отдаче/усилиям. Дата: 2026-07-16.

Сквозная тема: у **MCP-бэкенда** (`src/mcp/McpServer.cpp`, ~40 инструментов) уже
есть много силы, не выведенной в GUI (backtrace/frames, coverage, search/diff/type,
io-log/emt-log) — самый дешёвый источник фич. А самое ценное новое —
**обратная отладка**; архитектура с save/restore к ней уже готова.

## ✅ Сделано

- **Символы/метки в дизассемблере** — `bk::SymbolTable` (core), `Board::loadSymbols`,
  символ-aware `bk::disasm`, метки/цели/стек/PC в Soft-ICE оверлее, меню загрузки
  `.map`. (коммит по теме символов)
- **Подсветка изменившихся регистров** — `DebuggerOverlay::snapshotRegs()` + оранжевая
  подсветка изменённых R/PSW после шага.

## 🎯 Высокий приоритет

- **Обратная отладка / «машина времени».** Шаг назад, «выполнить назад до брейка».
  Дёшево: кольцо снапшотов состояния каждые N инструкций (save/restore уже есть) →
  шаг назад = восстановить ближайший снапшот и до-прогнать до target−1. Самая
  мощная фича, которой нет.
- **Панель стека вызовов (backtrace) в GUI.** `bk_backtrace`/`bk_frames` уже есть в
  MCP; оверлей показывает сырые слова стека — добавить разрешённую цепочку вызовов
  (адрес → имя+смещение через `Board::symbols()`).
- **Профайлер прямо в дизассемблере.** Рядом с каждой строкой: `% CPU-времени /
  такты / число исполнений` (Trace уже считает exec-count, есть `timingFor`; дизасм
  уже тепло-тонирует по частоте).

## 🔧 Дёшево, переиспользует существующее

- **Поиск и правка памяти в GUI.** MCP умеет `search/diff/write`; панель ПАМЯТЬ
  только читает — добавить find + правку по двойному клику.
- **Трейс-пойнты / лог-пойнты.** Не останавливающие «брейки», пишущие в лог
  (регистры/выражение) → история исполнения без остановки игры.
- **Панель покрытия кода.** `bk_coverage` есть; GUI-вид (как memvis, но по коду):
  что исполнялось / что ни разу → мёртвый код, непройденные ветки.
- **Плоская таблица профиля.** Сортируемый топ функций (self / inclusive / calls /
  avg), классический профайлерский вид (данные уже во flame-графе/Trace).
- **Хит-каунт брейки.** «Сработать на N-й раз / пропустить N» (у условных брейков
  уже есть инфраструктура `breakConds_`).

## 🖥 Специфика БК/PDP-11

- **Детектор самомодифицирующегося кода.** Подсветить исполнение байт, недавно
  записанных (Trace знает `lastWrite`/`lastExec`); частый приём в демках/защитах БК.
- **Таймлайн прерываний и I/O.** Визуализировать IRQ2 50 Гц, прерывание клавиатуры,
  EMT-вызовы, записи в I/O-регистры (`io-log`/`emt-log` уже в MCP).
- **Осциллограф звука.** Рисовать поток `Speaker` (1-битный бипер) как волну — отладка
  звука.
- **Растровая отладка видео.** Позиция луча + per-scanline скролл/режим — стыкуется с
  [[scanline-video-rendering]] (`docs/scanline-video-rendering.md`): «что на экране в
  такте X».
- **Отладка по исходнику.** У игр пользователя (репозиторий Digger, C/asm) есть
  исходники: по `.map`/листингу связать `адрес ↔ строка исходника`, брейки и шаг по
  строкам исходника. Сильно для рабочего процесса (символы уже загружаются —
  половина пути пройдена).

## ✨ Качество жизни

- **Выражения-watch** — вычислять `Rn`, `@addr`, слово/байт, биты PSW «на лету».
- **Комментарии/закладки** на адресах (сохраняемые), «типизированный» просмотр памяти
  (`bk_type` уже есть в MCP).
- **Диф двух сохранённых состояний** — что в памяти изменилось между моментами
  (`bk_diff_mem` есть в MCP).
- **Панель «Символы/метки»** с поиском и переходом (таблица уже загружается через
  `Board::symbols()`; осталась вёрстка панели — место в оверлее ограничено).
