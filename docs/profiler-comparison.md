# Сравнение профилировщика с другими и что ещё можно добавить

Обзор возможностей встроенного профилировщика эмулятора БК-0010 в сравнении с
известными инструментами (perf/VTune, **KCachegrind/Callgrind**, flame graphs
Б. Грегга, gprof, **beebperf**, отладчики эмуляторов) и список того, чего пока
не хватает.

## Что уже есть

- **Карта памяти** (`Ctrl+I`) — тепловая карта чтений/записей/исполнения с
  затуханием (green=read, red=write, blue=exec).
- **Горячий путь** (`Ctrl+G`) — топ-N самых исполняемых цепочек базовых блоков
  (трасс) со сворачиванием до ассемблерных инструкций, подсветка по «температуре».
- **Граф вызовов** (`Ctrl+K`) — в стиле KCachegrind: цвет=доля времени CPU,
  толщина стрелки=частота вызовов (лог), послойная раскладка с маршрутизацией
  рёбер между блоками, перетаскивание, вложенная «карта вызовов» (treemap),
  дизасм в боксах при зуме, затемнение простаивающих, топ-N.
- **Пламенный граф** (`Ctrl+F`) — flame graph по восстановленному на лету дереву
  вызовов (JSR/возвраты/прерывания), ширина ∝ inclusive-времени, drill-down.
- **Горячие инструкции во времени** (`Ctrl+H`) — временной ряд %CPU.
- **Подсветка тепла в дизассемблере отладчика** — фон инструкции по частоте.
- **Точные счётчики** (не сэмплинг) + оценка тактов (`Cpu::instrTicks`).

## Чего ещё не хватает (по приоритету)

| # | Функция | Откуда идея | Почему ценно для БК | Статус |
|---|---------|-------------|---------------------|--------|
| 1 | **Flame graph / icicle** | Грегг, все совр. | Реконструкция стека вызовов во времени; ширина=время, высота=глубина. | ✅ **сделано** (`Ctrl+F`) |
| 2 | **Бюджет кадра / тайминг игрового цикла** | beebperf | Для игр 50 Гц: сколько тактов из ~60000 съедено за кадр, график времени кадра, детект переполнения/джиттера и «разрыва». | ⬜ |
| 3 | **Self vs inclusive + панель Caller/Callee** | KCachegrind, gprof | Разделять собственную и накопленную стоимость; таблица «кто зовёт / что зовёт» с %. Inclusive уже считается для карты вызовов и пламенного графа. | ⬜ (частично) |
| 4 | **Атрибуция обращений к памяти** | beebperf | «какие адреса читаются/пишутся чаще, какими подпрограммами, какими инструкциями» — привязать heatmap к маршрутам. | ⬜ |
| 5 | **Диф-профилирование + экспорт CSV/буфер** | beebperf, VTune | Сравнение двух прогонов (до/после оптимизации) с зелёно-красными дельтами; выгрузка таблиц. | ⬜ |
| 6 | **Детект циклов (SCC) и их сворачивание** | KCachegrind | Группировать взаимную рекурсию (мы ломаем циклы для раскладки, но не сворачиваем). | ⬜ |
| 7 | **Гистограмма опкодов / режимов адресации / тактов по категориям** | — | Дёшево из `execCount`, полезно для оптимизации. | ⬜ |
| 8 | **Учёт прерываний (ISR)** | perf | Время в 50 Гц IRQ / клавиатурном ISR vs основной код (детект по векторам; в пламенном графе ISR уже выделяется отдельным кадром). | ⬜ (частично) |

## Приёмы визуализации, которые ещё не применяли

- **Спарклайны в таблицах** — мини-график активности функции в каждой строке
  хот-листа.
- **«Piano-roll» / тепловая полоса во времени** — активность адреса/функции как
  горизонтальная лента (когда именно исполнялось).
- **Бар бюджета кадра** — использование vs 50 Гц-бюджет, краснеет при переполнении.
- **Диф-раскраска** (зелёный/красный дельты между прогонами).
- **Связанная подсветка между окнами** — навёл на функцию → подсветилась в
  heatmap, графе, хот-пути (частично есть через `addressPicked`).
- **Иерархический edge-bundling** для графа вызовов — сгруппировать параллельные
  вертикальные рёбра, убрать «занавес» из линий.
- **Легенда цветовой шкалы** и **миникарта** для больших графов.
- **Семантический зум** — уже есть (дизасм в боксах при увеличении); можно
  распространить на другие окна.

## Наши преимущества (стоит подчёркивать)

- **Точный, а не сэмплинговый профиль** — можно показывать **точные суммарные
  такты** на инструкцию/строку, а не оценку, чего нет у сэмплинг-профайлеров.
- **Растровый тайминг** — на какой строке кадра исполняется код; прямой путь к
  анализу «разрывов» как в beebperf.

## Рекомендуемый порядок работ

1. **Бюджет кадра / тайминг игрового цикла** — быстро и максимально в тему для
   50 Гц-игр.
2. ~~Flame graph~~ — ✅ сделано.
3. **Панель caller/callee с self/inclusive** — паритет с KCachegrind.
4. **Атрибуция памяти** — связать существующий heatmap с маршрутами.
