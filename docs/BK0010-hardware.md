# Справочник по железу БК-0010-01 (для эмулятора)

Проверенные факты, использованные в эмуляторе. Значения — в **восьмеричной**
системе (как в документации БК и PDP-11). Источник истины по семантике —
референсный C-эмулятор `/home/prcoder/emulators/БК-0010-01/bk`.

## Процессор К1801ВМ1 (клон PDP-11)

- 8 регистров: `R0..R5` общие, `R6=SP`, `R7=PC`. PSW (слово состояния).
- Флаги PSW: `C=001`, `V=002`, `Z=004`, `N=010`, `T=020` (трассировка),
  бит приоритета/маски `0200`. При включении питания PSW = `0340`
  (приоритет 7, прерывания замаскированы) — монитор сам понижает приоритет
  после установки векторов.
- Набор: базовый PDP-11 + `SOB` + `XOR`. **Нет** `MUL/DIV/ASH/ASHC` (EIS) и FPU.
- Диспетчеризация: таблица по `ir >> 6` (1024 входа) + подтаблицы для группы
  `0000xx` (HALT/WAIT/RTI/BPT/IOT/RESET/RTT) и `0002xx` (RTS/CCC/SCC).
- Поля инструкции: `SRC_MODE=(ir>>9)&7`, `SRC_REG=(ir>>6)&7`,
  `DST_MODE=(ir>>3)&7`, `DST_REG=ir&7`. Режимы адресации 0..7:
  `Rn`, `(Rn)`, `(Rn)+`, `@(Rn)+`, `-(Rn)`, `@-(Rn)`, `d(Rn)`, `@d(Rn)`.
  Байтовые авто-инкремент/декремент для `R0..R5` — на 1, для `R6/R7` — на 2.
- CMP/SUB: флаг C **инвертирован** (C=0 при отсутствии заёма; для равных
  операндов C сбрасывается).

## Векторы прерываний

| Вектор | Назначение                        |
|--------|-----------------------------------|
| `004`  | Ошибка шины                       |
| `010`  | Зарезервированная инструкция      |
| `014`  | T-бит / BPT                       |
| `020`  | IOT                               |
| `024`  | Авария питания                    |
| `030`  | EMT                               |
| `034`  | TRAP                              |
| `060`  | Клавиатура                        |
| `100`  | IRQ2 — кадровое 50 Гц             |

Обработка trap/interrupt: push PSW, push PC, затем `PC=[vec]`, `PSW=[vec+2]`.
Старт БК-0010 при включении: `PC=0100000` (монитор).

## Карта памяти (64 КБ, слово-адресуемая)

| Диапазон          | Назначение                              |
|-------------------|------------------------------------------|
| `000000–037777`   | ОЗУ пользователя (16 КБ, вкл. системные ячейки и стек) |
| `040000–077777`   | Экранное ОЗУ (16 КБ), оно же обычное ОЗУ |
| `100000–117777`   | ПЗУ монитор `monit10.rom` (8 КБ)         |
| `120000–177577`   | ПЗУ Бейсик `basic10.rom` (24 КБ)         |
| `177600–177777`   | Страница регистров ввода-вывода          |

## Регистры ввода-вывода

| Адрес     | Регистр                                                    |
|-----------|------------------------------------------------------------|
| `177660`  | Состояние клавиатуры (бит 7 = данные готовы, бит 6 = маска прерывания) |
| `177662`  | Данные клавиатуры (чтение сбрасывает флаг готовности)      |
| `177664`  | Скролл (младший байт — номер строки; исходно `0330`)       |
| `177706`  | Таймер: предел                                             |
| `177710`  | Таймер: счётчик                                            |
| `177712`  | Таймер: управление (такт f/128 ≈ 23438 Гц)                 |
| `177714`  | Параллельный порт (принтер/джойстик/covox)                 |
| `177716`  | Системный: чтение — бит15=1, старший байт `0200`; запись — биты 6/7 = лента/пищалка |

## Программируемый таймер 1801ВМ1 (0177706 / 0177710 / 0177712)

Встроенный в процессор интервальный таймер. **Опрашивается программно — прерывание
не генерирует** (в отличие от кадрового 50 Гц). Реализация повторяет `bk/timer.c`.

- `0177706` — регистр предела (перезагрузки), чтение/запись.
- `0177710` — счётчик (только чтение; запись игнорируется). Убывает.
- `0177712` — регистр управления/состояния. Чтение возвращает `0177400 | control`.

Такт счётчика = f/128 = 3 МГц / 128 ≈ 23438 Гц (период 128 тактов ЦП), делится
дополнительно на 4 и/или 16 битами D4/D16.

Биты регистра управления `0177712`:
| Бит | Имя | Назначение |
|-----|-----|------------|
| 0 (`001`) | SP | Выбор тактирования (не используется) |
| 1 (`002`) | CAP | Непрерывный счёт **без** перезагрузки (свободный ход) |
| 2 (`004`) | MON | Разрешить установку флага события (FL) |
| 3 (`010`) | OS | Однократный режим (после нуля счёт останавливается) |
| 4 (`020`) | RUN | Разрешение счёта (пуск) |
| 5 (`040`) | D16 | Делитель на 16 |
| 6 (`100`) | D4 | Делитель на 4 |
| 7 (`200`) | FL | Флаг события (устанавливается при переходе через ноль) |

Режимы: RUN=0 — стоп; RUN=1,OS=0,CAP=0 — непрерывный с перезагрузкой из предела;
RUN=1,OS=1,CAP=0 — однократный; RUN=1,CAP=1 — непрерывный без перезагрузки.
При пуске (переход RUN 0→1) счётчик предзагружается значением предела. Запись
в регистр управления сбрасывает флаг FL, если бит 7 в записанном значении = 0.

## Экран

Экранное ОЗУ 16 КБ = **256 строк × 64 байта**. Смещение строки:
`memLine = (y + (scroll&0377) - 0330) & 0377`.

- **Цветной режим 256×256** (2 бита/пиксель): байт = 4 пикселя, младшими битами
  вперёд; значение 0..3 индексирует палитру. В эмуляторе пиксель дублируется по
  горизонтали до ширины 512.
- **Ч/б режим 512×256** (1 бит/пиксель): байт = 8 пикселей, младшим битом вперёд;
  0/1 → чёрный/белый.
- Палитра 0 (аппаратный порядок, индексы 0..3): **{чёрный, синий, зелёный, красный}**.
  В «бейсиковом» порядке цвета 1 и 3 меняются местами. Всего 16 палитр.
- Формат текстуры в эмуляторе: `0xAARRGGBB` (uint32), в GL грузится как `GL_BGRA`,
  в Qt — `QImage::Format_ARGB32`.

## Формат файла `.BIN`

Заголовок 4 байта (little-endian): `[адрес загрузки:2][длина в байтах:2]`, далее
сырые данные. Для запуска: загрузить по адресу, установить `PC = адрес загрузки`.
**Важно:** сначала должен отработать монитор из ПЗУ (инициализация векторов,
стека, драйвера дисплея), только потом переход на игру — иначе кадровое прерывание
уйдёт в неустановленный вектор и процессор остановится (HALT по адресу 0).

## Тактирование

CPU 3 МГц, кадр 50 Гц → 60000 тактов/кадр. Такты инструкций — по таблицам из
`bk/timing.c` (базовая рег-рег ≈ 12 тактов + добавки по режимам адресации; ветвления
≈16, JSR ≈32–48, EMT/TRAP ≈68).

## Клавиатура (КОИ-7)

Регистры: `177660` — состояние (бит 6 = разрешение прерывания, бит 7 = «код готов»),
`177662` — данные (младшие 7 бит кода; чтение сбрасывает бит готовности), `177716`
бит 6 — индикатор нажатия (активный 0). Вектор прерывания клавиатуры: `060` для
обычных кодов, `0274` для функциональных клавиш/АР2 (код с установленным битом `0200`).

**Регистр хранит ровно один код** (как в реальном БК-0010): новый код принимается,
только пока бит «код готов» сброшен (предыдущий код уже прочитан); иначе нажатие
теряется. Чтение `177662` сбрасывает бит готовности и снимает индикатор нажатия.

Коды клавиш (проверено по эталонному эмулятору bkbtl):
- Печатные ASCII 0x20..0x7E (латиница/цифры/пунктуация) — как есть (КОИ-7 Н0).
- ВВОД=`012`, ЗАБ(Backspace)=`030`, Tab=`015`, Пробел=`040`, ВС(Home)=`023`,
  СБР(Delete)=`014`.
- Стрелки: влево=`010`, вправо=`031`, вверх=`032`, вниз=`033`.
- Функциональные: ПОВТ(F1)=`0201`, КТ(F2)=`0003`, `0231`(F3), `0026`(F4),
  `0027`(F5), ИНД СУ(F6)=`0202` (коды с битом `0200` → вектор `0274`).
- СУ(Ctrl)+буква → управляющий код `001..032`.
- Кириллица — КОИ-7 Н1, коды `0140..0177` (порядок: ЮАБЦДЕФГХИЙКЛМНОПЯРСТУЖВЬЫЗШЭЩЧЪ).
- Переключение регистра РУС/ЛАТ — коды `016`(РУС, SO) / `017`(ЛАТ, SI). Эмулятор
  вставляет их автоматически при смене языка ввода (латиница ↔ кириллица),
  имитируя нажатие клавиши РУС/ЛАТ.
